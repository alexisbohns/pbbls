### Task 8: Size-proportional pebble render in detail & edit sheets

**Context:** Both `PebbleDetailSheet` and `EditPebbleSheet` (via `PebbleFormView`) render the composed pebble SVG inside a fixed `260pt`-tall frame. Small pebbles fill that frame aspect-to-fit, which makes them *appear* as big as — or bigger than — medium and large pebbles. Scale the render height by the pebble's `Valence.sizeGroup`: small = `180`, medium = `220`, large = `260`. `PebbleDetail.valence.sizeGroup` gives us the mapping; `EditPebbleSheet` already decodes the detail, so it can pass the height down through `PebbleFormView`.

**Files:**
- Modify: `apps/ios/Pebbles/Features/Path/PebbleFormView.swift` (add `renderHeight` prop, use it on the render frame around line 31)
- Modify: `apps/ios/Pebbles/Features/Path/EditPebbleSheet.swift` (store the size group; pass `renderHeight` to `PebbleFormView`)
- Modify: `apps/ios/Pebbles/Features/Path/PebbleDetailSheet.swift` (use the size group to size the render frame around line 56)

- [ ] **Step 1: Define the shared height helper on `ValenceSizeGroup`**

Edit `apps/ios/Pebbles/Features/Path/Models/Valence.swift`. At the end of the `extension Valence { ... }` block (or as a new extension on `ValenceSizeGroup`), append:

```swift
extension ValenceSizeGroup {
    /// Render height in the detail / edit sheets. Scales small pebbles down
    /// so a small render doesn't visually dominate a medium or large one —
    /// addresses issue #286 "small pebbles are full width, they should be a
    /// little bit smaller".
    var renderHeight: CGFloat {
        switch self {
        case .small:  return 180
        case .medium: return 220
        case .large:  return 260
        }
    }
}
```
