# Android Lab ‚Äî feeds, reactions, announcement detail, Profile entry

> Milestone **M44 ¬∑ Android Lab** (issues #588‚Äì#591). Reference: the parity audit ("Lab feature", ~1,060 LOC) and iOS sources under `apps/ios/Pebbles/Features/Lab/`. Zero Android Lab code exists today; the Profile Lab card was deliberately omitted in M41 (D11 no-dead-chrome) and un-hides here.

## Goal

Android users can open the Lab from their Profile and see what the maintainer shipped, is building, and has queued ‚Äî announcements (with a markdown detail page), changelog, in-progress initiatives, and an upvotable backlog ‚Äî plus the WhatsApp community card. Exit: Lab loads all four feeds; tapping an announcement opens its rendered body; upvoting a backlog row moves its count instantly and survives an error by reverting; "See all" lists page the full changelog/backlog; fr pass.

## Non-goals

Publishing to `logs` from any dev loop (human-only via the Lab admin ‚Äî repo rule); markdown beyond the iOS V1 subset (match, don't exceed); platform filtering client-side (iOS decodes `platform` and never uses it); push/notification hooks for new entries; reaction counts anywhere outside the Lab.

## Scope ‚Äî sub-projects

- **A (#589)** data layer: `Log` model + enums, lossy list decode, `LogsService` (4 feeds + `myReactions` + `react`/`unreact`), JVM test suite. Parallel-safe with nothing ‚Äî it's first.
- **B (#590)** Lab screen: `LabScreen` (5 concurrent fetches), `LogTimeline`, `ReactionButton`, `AnnouncementRow`, `FeaturedCommunityCard`. Blocked by A.
- **C (#591)** announcement detail (markdown subset), see-all `LogListScreen`, `ProfileLabCard` entry, Arkaik. Blocked by A and B (reuses the timeline + reaction button).

## Core design decisions

- **D1 ‚Äî Unknown `platform` rows are dropped, matching iOS (the decision #588 demands).** `logs.platform`'s check constraint admits `project`/`infra`, but iOS's strict enum makes the lossy decoder skip those rows silently. Android ports the same behavior **deliberately**: `LogPlatform` enumerates exactly `webapp | ios | android | all`, and a `project`/`infra` row is dropped by the lossy decode like any other bad row. Rationale: the two mobile surfaces must show the same feed content; `project`/`infra` entries are internal bookkeeping the maintainer never intended for the app feed; and rendering rows iOS hides would diverge user-visible content between platforms. A dedicated JVM test pins the drop (`platform: "project"` ‚Üí row skipped, siblings kept). If these rows should ever surface, both apps change together.
- **D2 ‚Äî Lossy decode is per-element, hand-rolled, and never coercing.** Feeds decode the response as a `JsonArray` first, then map each element to `Log` catching per-element failures ‚Äî one error log per skipped row (tag `logs-service`, index + cause), happy path silent. No `coerceInputValues`, no fallback enum case, no defaulting: any bad element (unknown enum, malformed UUID, missing required key) drops the **whole row**. A non-array top level still throws so screens reach their error state. The iOS 8-test suite ports 1:1 (all-valid order, empty array, mid-array bad row, consecutive bad rows, all-bad ‚Üí empty not throw, non-array throws, `released_at` present/absent) plus the D1 platform test.
- **D3 ‚Äî Feed queries are the four iOS predicates verbatim on `v_logs_with_counts`; the Lab screen fires 5 concurrent fetches that fail independently.** announcements: `species=announcement, published=true, order published_at desc`; changelog: `species=feature, status=shipped, published=true, order released_at desc NULLS LAST, then published_at desc`; initiatives: `species=feature, status=in_progress, published=true, order published_at desc`; backlog: `species=feature, status=backlog, published=true, order reaction_count desc, then created_at desc`. Lab screen: `feedLimit = 5` on changelog + backlog only; announcements/initiatives unlimited. Load = `coroutineScope { async√ó5 }`, each caught to empty + logged; the fullscreen "Couldn't load the Lab." fires **only when all four content feeds fail** (a reactions-only failure just yields an empty reacted set). `LogListScreen` is the opposite: any failure (feed **or** reactions) ‚Üí "Couldn't load the list.". `myReactions()` returns an empty set without a session (no throw).
- **D4 ‚Äî Reactions write straight to `log_reactions`; deliberately no RPC.** Single-table, single-statement insert/delete on PK `(log_id, user_id)` ‚Äî the sanctioned direct-write pattern per root `AGENTS.md` (iOS comments the same). `react` inserts `{log_id, user_id}`, `unreact` deletes by both columns; both throw missing-session ("Please sign in again.") and rethrow network errors. Optimistic toggle (identical in Lab and see-all): flip membership in the `reactedIds` set + swap the one log with `withAdjustedCount(¬±1)` (clamped ‚â• 0), await the write, and on failure revert **both** exactly ‚Äî no server refetch, reconciliation is purely revert-on-error. A duplicate-insert PK conflict surfaces as an error and reverts like any failure.
- **D5 ‚Äî The announcement body renders the iOS V1 markdown subset ‚Äî match, don't exceed.** A pure `LabMarkdown` parser (JVM-tested): split on literal `\n\n`, trim, drop empties; `### `/`## `/`# ` prefixes ‚Üí heading blocks whose text stays **plain** (no inline parsing); everything else is a paragraph with inline-only parsing ‚Äî `**bold**`, `*italic*`, `` `code` ``, `[link](url)`, `~strikethrough~` ‚Äî where single newlines stay visible line breaks. Lists, blockquotes, images, tables, fenced code render as literal paragraph text. Parse failure ‚Üí raw string verbatim. Compose rendering via `AnnotatedString` + `PebblesText` (headings 28/22/20 bold foreground, paragraphs body foreground); links open externally.
- **D6 ‚Äî The Lab-only error strings stay English in both locales (named iOS quirk).** iOS assigns "Couldn't load the Lab." / "Couldn't load the list." as raw strings, bypassing its catalog ‚Äî they render English even in French. Android keeps resource discipline (keys in both `strings.xml` files so `LocalizationParityTest` passes) but the fr values carry the same English copy, with a comment naming the parity quirk. All other Lab copy uses the real iOS FR catalog values ‚Äî including the reaction button's overridden a11y strings (reacted label EN "Remove rock" / FR "√îter mon vote", value format EN "%d upvotes" / FR "%d rocks" ‚Äî displayed values, not the source keys).
- **D7 ‚Äî Cover images come from the public `lab-assets` bucket; no signing.** `LabConfig` holds `assetsBucket = "lab-assets"` and the WhatsApp invite URL; cover URL = `{supabaseURL}/storage/v1/object/public/lab-assets/{path}` only when `cover_image_path` is non-null **and** non-empty, loaded with Coil `AsyncImage` (muted-30% rounded placeholder, 140dp rows / 200dp detail, cover-crop). `supabaseURL` reaches the helper through `AppEnvironment` (D8 secrets flow), not a hardcoded host.
- **D8 ‚Äî The WhatsApp card is external-link chrome, always first, never gated.** `FeaturedCommunityCard` renders above the feeds regardless of feed outcomes: icon + "Join the community" + footnote, full-width filled accent button "Open in WhatsApp" firing `ACTION_VIEW` on the invite URL (no in-app webview). FR: "Rejoindre la communaut√©" / "Aide-nous √Ý fa√ßonner Pebbles sur WhatsApp." / "Ouvrir dans WhatsApp".
- **D9 ‚Äî Lab is one pushed route; detail and see-all are conditionally composed sub-surfaces inside it.** `ROUTE_LAB` pushes from `ProfileLabCard` (M41 D1 push discipline). Within `LabScreen`, the announcement detail and the two `LogListScreen` modes are conditional full-content swaps with `BackHandler` (the `PathScreen` cover precedent) rather than NavHost sub-routes ‚Äî the selected `Log` object passes directly, avoiding a nonexistent single-log fetch and route-argument serialization. Timeline geometry ports as specified (16dp icon column, 1dp muted connector, transparent first lead-in, changelog date = `released_at ?? published_at` locale-aware long date via the active `Locale`).
- **D10 ‚Äî `LogsService` is a D4-pattern plain class; screens own their feed state.** Constructor takes `SupabaseService`; provided via `LocalLogsService`. Feed lists/reacted set live in screen state (iOS keeps them in the views), so previews drive pure content composables (`LabContent`, `LogTimeline`, `AnnouncementDetailContent`) with fixture data ‚Äî the screenshot gallery covers timeline modes, reaction states, announcement rows, the WhatsApp card, and the markdown subset.
- **D11 ‚Äî `ProfileLabCard` un-hides on Profile after the Collections card, before sign-out.** Lightbulb icon (accent), "Lab" headline, subtitle "News & community" (FR "Actus & communaut√©" ‚Äî not "Lab news"), muted chevron, `profileCard` chrome (17dp padding/radius, 1dp muted border, transparent fill). It was the last deliberately-hidden M41 tile; Profile's no-dead-chrome rule is satisfied the moment the route exists (same PR).

## Risks

1. Markdown subset drift ‚Äî mitigated: pure parser with JVM tests transcribed from the iOS block/inline rules before any UI lands; unsupported constructs assert literal rendering.
2. Lossy-decode parity ‚Äî mitigated: the ported 8-test suite + the D1 platform test gate sub-project A's merge.
3. `NULLS LAST` ordering through supabase-kt's PostgREST builder ‚Äî verify the generated query orders `released_at` descending with nulls last (`order(..., nullsFirst = false)`); if the builder can't express it, order client-side after decode and document it in the service.
